`.daemon8-cli.toml` is the per-project config the CLI hook reads. Create one with `daemon8 init` at your project root — the command also walks you through hook registration, see <Link to="/docs/$slug" params={{ slug: "enroll-your-agent" }}>Enroll Your Agent</Link>. The file lives next to `.gitignore` and `.editorconfig`.

## Resolution

Values are merged last-wins:

1. Compiled-in defaults
2. User file: `cli.toml` in the daemon8 config directory (same location as `config.toml`)
3. Project file: the nearest `.daemon8-cli.toml` found by walking up from the current working directory, stopping at the first `.git` directory or the filesystem root

Parse errors in either file are absorbed and the handler continues with the next lower layer.

## Minimal example

`daemon8 init` writes this template. Copy it verbatim if you are writing the file by hand:

```toml
# Daemon8 CLI hook configuration.
# Schema reference: https://daemon8.ai/docs/cli-hook-config

[project]
slug = "my-project"
role_default = "solo"

[enrollment]
enabled = true
scope = []

[features]
intents = true
inbox = true
state_tracking = true
compaction_recovery = true
heartbeat_interval = "30s"

[intents.auto_declare]
expertise = []

[distillation]
track_todowrite = true
track_git_commits = true
track_file_writes = true
coarsen_file_writes_below_threshold = 5
```

  ]}

`role_default` is a project-declared label attached to this agent when it joins the stream. Roles are how specialist agents discover each other and coordinate — the daemon does not enforce semantics on them. `watchdog` is the one role the hook treats specially: watchdog agents advertise read-only capabilities in their join card.

## `[providers.*]`

Per-CLI overrides keyed by tool name (e.g. `[providers.claude-code]`, `[providers.copilot]`).

<SimpleTable
  headers={["Key", "Type", "Default", "Notes"]}
  colWidths={["220px", "120px", "140px", "auto"]}
  rows={[
    ["enabled", "bool", "true", "When false, the hook skips enrollment when invoked under this tool. A provider-level false always overrides [enrollment].enabled = true."],
    ["reason", "string", "(none)", "Human-readable explanation kept alongside the override."],
  ]}
/>

Example — disable enrollment under Copilot only:

```toml
[providers.copilot]
enabled = false
reason = "internal policy"
```

## `[sources]`

A pass-through table for arbitrary project metadata (data source URLs, owner tags, anything else you want attached to this project's config). The hook does not interpret `[sources]`; downstream tooling may.
