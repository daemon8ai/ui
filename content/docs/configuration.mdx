The daemon reads a TOML config at startup. For the 99% case, `daemon8 setup` writes a working file and you never touch it. When you do need to change something, the key reference is below.

Print the path on your machine with `daemon8 config path`. Typical locations:

- macOS: `~/Library/Application Support/dev.daemon8.daemon8/config.toml`
- Linux: `~/.config/daemon8/config.toml`
- Windows: `%APPDATA%\daemon8\daemon8\config.toml`

## Resolution order

Values are merged last-wins:

1. Compiled-in defaults
2. The TOML file at the path above (or the file passed to `--config`)
3. Environment variables with the `DAEMON8_` prefix, using `__` for nesting

Example env override: `DAEMON8_SERVER__PORT=9090 daemon8 serve` starts on port 9090 regardless of the file.

## Changing config

The most common workflow is `daemon8 config set`, which validates the value and writes the key atomically:

```bash
daemon8 config set browser.endpoint http://localhost:9333
```

`daemon8 config show` prints the resolved configuration. Pass `--json` for machine-readable output. Most keys require restarting the daemon to take effect; the CLI prints a reminder when it detects that the daemon is running.

<Callout>
  Prefer <code>daemon8 setup</code> for first-time configuration. It handles browser detection, MCP client wiring, and service install in one pass.
</Callout>
