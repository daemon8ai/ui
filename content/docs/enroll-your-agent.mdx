Enrollment makes your agent's sessions visible on the stream. When a session starts, it announces itself. When it calls a tool, it emits a heartbeat. When it ends, it says goodbye. Other agents — or you, from another terminal — can see all of it in real time.

Two steps: create a project config, then register the hooks in your CLI.

## 1. Create the project config

At the root of the project you want to enroll:

```bash
daemon8 init
```

This writes `.daemon8-cli.toml` using the directory name as the project slug and `solo` as the default role. Override either with flags:

```bash
daemon8 init --slug my-project --role worker
```

Pass `--force` to overwrite an existing config. The four roles are `solo` (single agent), `queen` and `worker` (coordinated multi-agent), and `watchdog` (observe-only).

## 2. Register hooks in your CLI

Your AI tool needs to call `daemon8 cli-hook` at key moments during a session. Each tool has its own settings format — pick yours below.

### Claude Code

Add a `hooks` block to `.claude/settings.local.json` in the project directory (or `~/.claude/settings.json` for every project on this machine):

```json
{
  "hooks": {
    "SessionStart": [{ "type": "command", "command": "daemon8 cli-hook" }],
    "Stop": [{ "type": "command", "command": "daemon8 cli-hook" }],
    "PreToolUse": [{ "type": "command", "command": "daemon8 cli-hook" }],
    "PostToolUse": [{ "type": "command", "command": "daemon8 cli-hook" }],
    "PreCompact": [{ "type": "command", "command": "daemon8 cli-hook" }],
    "UserPromptSubmit": [{ "type": "command", "command": "daemon8 cli-hook" }]
  }
}
```

Every event uses the same command. The CLI passes event data on stdin; the handler normalizes it automatically.

### Other tools

Cursor, Gemini CLI, Codex, Copilot, Continue, and opencode are all supported. Register `daemon8 cli-hook` in each tool's native hook mechanism for the equivalent events (session start/end, tool use, compaction). The handler detects the tool from environment variables — no `--tool` flag needed for most. See <Link to="/docs/$slug" params={{ slug: "cli-hook" }}>CLI Hook</Link> for the env vars each tool sets.

## Verify

Open a new session in the enrolled project. The `SessionStart` hook fires when the session boots and the first observation lands on the stream within a second or two.

Ask your agent:

```text
query_observations with origins=["app:daemon8-cli-hook"]
```

You should see an `agent.joined` observation:

```json
{
  "origin": { "type": "application", "name": "daemon8-cli-hook" },
  "kind":   { "type": "custom", "channel": "agent" },
  "severity": "info",
  "data": {
    "event": "agent.joined",
    "agent_id": "your-host:claude-code:1a2b3c",
    "project_slug": "my-project",
    "role": "solo",
    "tool": "claude-code",
    "model": "claude-opus-4-7"
  }
}
```

If you see that, you're enrolled. Tool calls emit `agent.heartbeat`; session end emits `agent.departed`.

## What each event becomes

The handler normalizes event names across all supported tools into a small set of stream observations:

    ["SessionEnd / Stop", "agent.departed", "Agent is leaving the stream."],
    ["PreToolUse / UserPromptSubmit", "agent.heartbeat", "Agent is alive and working."],
    ["PostToolUse", "agent.heartbeat", "Agent completed a tool call."],
    ["PreCompact", "agent.precompact", "Context window is about to be compacted."],
    ["PostCompact", "agent.postcompact", "Compaction finished; agent is reorienting."],
  ]}

Unknown events are silently dropped. The hook runs with a short timeout and never blocks your CLI — if the daemon is unreachable, the hook exits silently.

## What's next

**Coordinate with other agents.** Every enrolled session announces its role. See <Link to="/docs/$slug" params={{ slug: "multi-agent-coordination" }}>Multi-Agent Coordination</Link> for the patterns — filtering on role, narrowing per-agent slices with `subscribe_observations`.

**Tune what gets emitted.** The defaults are sensible, but `.daemon8-cli.toml` has knobs for intent matching, distillation rules, per-tool overrides, and heartbeat cadence. See <Link to="/docs/$slug" params={{ slug: "cli-hook-config" }}>CLI Hook Config</Link>.

**Enroll another project.** Same two steps. `cd` into the project root, run `daemon8 init`, add the hooks block to that project's settings.

## Troubleshooting

- **`agent.joined` does not arrive.** Run `daemon8 status`. If the daemon is not running, hooks fire but have nowhere to post. Start it with `daemon8 install` or `daemon8 serve`.
- **No hook fires at all.** The session must be started *after* the hooks were registered. Close and reopen your CLI.
- **You moved the `daemon8` binary.** The hook command references `daemon8` by name, so it must be on your `PATH`. If you installed via `cargo install`, it's at `~/.cargo/bin/daemon8` — confirm that directory is in your shell's `PATH`.
