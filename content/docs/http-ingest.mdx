HTTP ingest is how application runtime writes into the stream. Anything that can send JSON over localhost can push an observation in — SDKs, curl, shell scripts, log shippers, custom workers. The default endpoint is `http://localhost:9077/ingest`. For batches, `POST /ingest/batch`.

## Minimum useful payload

```json
{
  "kind": "log",
  "app": "checkout",
  "severity": "info",
  "data": { "message": "payment attempt started" }
}
```

The daemon stamps its own `timestamp_ns` and assigns a monotonic `id` on insert. Clients cannot set either.

## Kinds

Recognized `kind` values map to typed observations — for example, `query` expects `data.sql` and `data.duration_ms`, `http` expects `data.method` and `data.url`, `metric` expects `data.name` and `data.value`. When the expected fields are missing, the observation is still stored, but as `custom` with the kind name as its channel. The full enum, including shapes for `js_exception`, `lifecycle`, and `state_snapshot`, is in <Link to="/docs/$slug" params={{ slug: "observation-schema" }}>Observation Schema</Link>.

## Channels

The `channel` field groups custom observations so agents can filter on them. Pair it with `kind: "custom"` — or omit `kind` entirely and the daemon infers custom from the presence of a channel. Channels are freeform strings; pick stable identifiers and reuse them across the lifetime of the feature producing them.

## Beyond HTTP

Two other ingest transports are available when enabled in <Link to="/docs/$slug" params={{ slug: "configuration" }}>configuration</Link>:

- UDP listener (default bind `127.0.0.1:9078`, max packet 65536 bytes). Off by default. Useful for fire-and-forget pipelines where blocking on an HTTP response is the wrong tradeoff.
- Unix socket (default path `/tmp/daemon8.sock`, Unix only). Off by default. Useful for on-host workers that want to avoid even the loopback cost.

Both carry the same JSON shape as HTTP. Payloads flow through the same normalization.

## What happens next

The observation lands in the store with an assigned id and timestamp. From that moment it is queryable via <Link to="/docs/$slug" params={{ slug: "mcp-tools" }}>debug_observe</Link>, counted by `debug_summary`, and pushed to any `debug_subscribe` session whose filter matches — including the default `severity >= warn` subscription every fresh session opens with.
