`issue_command` sends one command to a connected target. Browser commands route over CDP. Device screenshots route over ADB when `device_serial` is present. Every action ships with every install.

Browser actions take an optional `tab_id`. Omit it to target the first tab the daemon finds. Call the `list_tabs` action first if you need a specific one.

## eval_js

```json
{ "action": "eval_js", "expression": "document.title" }
```

Response: `{ "result": "Daemon8 - Runtime Observation" }`.

## screenshot

```json
{ "action": "screenshot", "selector": "header" }
```

Response: `{ "screenshot": "/path/to/file.png", "size_bytes": 48231, "selector": "header" }`. With `device_serial`, the response also carries `source` and `serial`.

## get_dom

```json
{ "action": "get_dom", "selector": "main" }
```

Response: `{ "html": "<main>...</main>" }`.

## navigate

```json
{ "action": "navigate", "url": "https://example.com" }
```

Response: `{ "navigated": true, "title": "Example Domain" }`.

## inject_css

```json
{ "action": "inject_css", "css": "body { outline: 2px solid magenta; }" }
```

Response: `{ "injected": true, "element_id": 42, "temporary": true }`.

## revert_css

```json
{ "action": "revert_css" }
```

Response: `{ "reverted_count": 3 }`.

## set_viewport

```json
{
  "action": "set_viewport",
  "viewport_width": 412,
  "viewport_height": 915,
  "viewport_scale": 2.625,
  "viewport_mobile": true
}
```

Response: `{ "viewport_set": true, "width": 412, "height": 915, "scale": 2.625, "mobile": true }`.

## clear_viewport

```json
{ "action": "clear_viewport" }
```

Response: `{ "viewport_cleared": true }`.

## get_perf_metrics

```json
{ "action": "get_perf_metrics" }
```

Response: `{ "metrics": { "...": "..." } }`.

## list_tabs

```json
{ "action": "list_tabs" }
```

Response: `{ "tabs": [{ "id": "...", "url": "...", "title": "..." }] }`.

## network_conditions

```json
{ "action": "network_conditions", "network_preset": "slow-3g" }
```

Response: `{ "network_conditions": "slow-3g" }`.

## storage_inspect

```json
{ "action": "storage_inspect" }
```

Response: an object describing cookies, localStorage, and sessionStorage for the tab origin.

## storage_set

```json
{
  "action": "storage_set",
  "store_type": "localstorage",
  "storage_key": "feature_flag",
  "storage_value": "on"
}
```

Response: `{ "set": true }`.

## storage_clear

```json
{ "action": "storage_clear", "storage_types": "cookies,local_storage" }
```

Response: `{ "cleared": true }`.

## element_at_point

```json
{ "action": "element_at_point", "x": 120, "y": 48 }
```

Response: the element descriptor at those coordinates.

## new_tab

```json
{ "action": "new_tab", "url": "https://example.com" }
```

Response: `{ "tab_id": "ABCDEF1234567890" }`.

## close_tab

```json
{ "action": "close_tab", "tab_id": "ABCDEF1234567890" }
```

Response: `{ "closed": true }`.

## Typical call sequence

```text
1. issue_command list_tabs                 find the tab_id
2. create_checkpoint                        mark the before state
3. issue_command navigate { tab_id, url }  trigger the flow
4. query_observations { since_checkpoint }  read the resulting stream delta
5. issue_command screenshot { tab_id }     capture the current state
```
