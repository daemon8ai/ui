An observation is one structured event in the stream. Every source produces the same shape: browser events from CDP, device logs from ADB, application runtime over HTTP ingest, and notes written through `ingest_observation`.

The shape is defined by the `Observation` struct and its three enums: `Origin`, `ObservationKind`, and `Severity`. The daemon stores observations in this shape and returns them in this shape.

Filters compare by level, so `severity_min = "warn"` matches both `warn` and `error`.

## Origin

`Origin` is a tagged union with three variants.

```json
{ "type": "application", "name": "checkout-api" }
{ "type": "browser",     "tab_id": "tab-abc", "url": "https://example.com/cart" }
{ "type": "device",      "serial": "emulator-5554", "platform": "android" }
```

`Application` carries only the app name you configured in the SDK. `Browser` carries the CDP tab id plus the page URL at event time. `Device` carries the ADB serial plus the detected `platform`, which is either `android` or `vega` (Amazon Fire TV). See <Link to="/docs/$slug" params={{ slug: "devices" }}>Devices</Link> for how platform detection works.

On the wire, kinds serialize with a `type` tag:

```json
{ "type": "http_exchange", "method": "GET", "url": "/api/users", "status": 200, "duration_ms": 42.0 }
{ "type": "query", "sql": "SELECT 1", "duration_ms": 3.5 }
{ "type": "lifecycle", "event_name": "ready", "frame_id": "frame-1" }
```

Fields declared on the variant are part of `kind`. Anything else you want to attach goes in `data`.

## Checkpoints

```rust
pub struct Checkpoint(pub u64);
```

A checkpoint is a monotonic sequence marker — nothing more than the highest observation id the store has handed out. `create_checkpoint` returns one, and `query_observations` accepts `since_checkpoint` to fetch only observations newer than that marker. Checkpoints are the resume mechanism for incremental queries.

<Callout>
The `id` on `Observation` and the `u64` inside `Checkpoint` are the same number space. An observation with `id = 42` is covered by a checkpoint of `42` or higher.
</Callout>
