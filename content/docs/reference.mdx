The `daemon8` binary manages the local service, exposes the stream, and drives the browser. Most ongoing agent work happens through <Link to="/docs/$slug" params={{ slug: "mcp-tools" }}>MCP</Link>. Reach for the CLI to manage the service, run ad-hoc queries, or drive the browser from a shell.

## Service lifecycle

  headers={["Command", "What it does"]}

## Setup

<SimpleTable
  headers={["Command", "What it does"]}
  colWidths={["260px", "auto"]}
  rows={[
    [<code key="setup">daemon8 setup</code>, "Interactive setup wizard. Picks a browser, writes MCP client config for detected tools, installs the service, and runs a health check. Run again any time to reconfigure."],
    [<code key="doctor">daemon8 doctor [--fix]</code>, "Diagnose common environment and configuration issues. Pass --fix to attempt automatic repairs."],
    [<code key="init">daemon8 init [--role ROLE] [--slug SLUG] [--force] [-y|--yes] [--install-hooks local|shared|global] [--force-hooks]</code>, <>Interactive wizard. Walks you through slug and role, writes <code key="f">.daemon8-cli.toml</code>, and optionally registers hooks in your CLI's settings. With <code key="y">--yes</code> it skips the wizard and accepts defaults; hook registration stays opt-in via <code key="i">--install-hooks</code>. See <Link to="/docs/$slug" params={{ slug: "enroll-your-agent" }}>Enroll Your Agent</Link> and <Link to="/docs/$slug" params={{ slug: "cli-hook-config" }}>CLI Hook Config</Link>.</>],

  ]}
/>

The `--role` flag accepts `queen`, `worker`, `solo`, or `watchdog`. The default is `solo`.

## Global flags

`--config PATH` points at a specific config file instead of the default location. `--verbose` (or `-v`) raises the log level to debug for the current invocation. Both apply to every subcommand.
